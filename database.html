Key¶

class ndns.key.Key(**kwargs)¶

Bases: sqlalchemy.ext.declarative.api.Base

Key abstraction:

CREATE TABLE keys (
    id INTEGER NOT NULL,
    zone_id INTEGER,
    name BLOB,
    parent_zone BLOB,
    key_type VARCHAR(3) NOT NULL, /* "KSK", "ZKS", "DZSK" */
    rrset_id INTEGER,
    PRIMARY KEY (id),
    FOREIGN KEY(zone_id) REFERENCES zones (id) ON DELETE CASCADE ON UPDATE CASCADE,
    CHECK (key_type IN ('KSK', 'ZKS', 'DZSK'))
);
CREATE INDEX ix_keys_zone_id ON keys (zone_id);
CREATE UNIQUE INDEX ix_keys_name ON keys (name);

In addition to database entry, this class manages RSA keys (private and public) on disk: several methods require explicit location of the folder where private/public key are stored or should be stored.

Note that the current implementation does not provide any protection of private keys and they are stored in non-encrypted form.

Whenever requested, the key (ndn.Key) will be loaded and cached.

Variables:
  • rrset – One-to-one relationship (shortcut) to ndns.rrset.RRSet data
  • zone – Back-reference to the ndns.zone.Zone to which the key belongs
